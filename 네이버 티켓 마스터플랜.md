### Claude — 텔레그램 원클릭 발권 계획 (2026-03-09)

**핵심 아이디어:**
텔레그램 주문 알림 메시지를 파싱 → 주문 정보 자동 추출 → 티켓 생성 API 호출 → 티켓 링크 즉시 반환

**파싱해야 할 정보:**
- 공연명: `[광주] MelON(멜론) 지브리 & 뮤지컬 오케스트라 콘서트 비지정석 광주`
- 좌석 등급: `VIP석`
- 매수: `3매`
- 구매자명: `문혜주`
- 연락처: `010-8738-4492`
- 주문번호: `2026030820377521`

**구현 방식 (2가지 선택지):**

#### 방식 A. 텔레그램 봇 커맨드 (추천)
- 텔레그램 봇이 주문 알림 메시지를 자동 감지하거나, 운영자가 메시지를 봇에게 forward
- 봇이 메시지를 파싱 → `createNaverOrderHttp` API 호출
- 생성된 티켓 링크를 텔레그램으로 바로 회신
- **장점:** 텔레그램 안에서 모든 게 끝남, 어드민 접속 불필요

#### 방식 B. 어드민 빠른 등록
- 어드민에 "주문 메시지 붙여넣기" 입력창 추가
- 텔레그램 메시지 텍스트를 그대로 붙여넣으면 자동 파싱
- 파싱 결과 확인 → 원클릭 발권
- **장점:** 파싱 결과를 눈으로 확인 후 발권 가능

**필요한 작업:**

1. **메시지 파서 구현** — 텔레그램 알림 형식에서 공연명, 등급, 매수, 구매자, 연락처, 주문번호 추출
2. **공연(이벤트) 매칭** — 파싱된 공연명으로 Firestore에서 해당 이벤트 자동 검색
3. **티켓 자동 생성** — 기존 `createNaverOrderInternal` 로직 활용, 파싱된 정보로 주문+티켓 한번에 생성
4. **티켓 링크 반환** — 생성된 모바일 티켓 URL을 즉시 돌려줌

**현재 이미 있는 기반:**
- `createNaverOrderInternal` (주문 생성 + 티켓 발급 로직)
- `createNaverOrderHttp` (HTTP 엔드포인트)
- 텔레그램 봇 (`telegram-command-bot.js`)
- 뿌리오 문자 발송 연동

**예상 흐름:**
```
텔레그램 알림 수신
  → 봇이 파싱 (또는 어드민에서 붙여넣기)
  → { eventId, buyerName, buyerPhone, orderNumber, seatGrade: "VIP", quantity: 3 }
  → createNaverOrderHttp 호출
  → 티켓 3장 생성 + 모바일 티켓 URL 3개 반환
  → 텔레그램으로 링크 회신 (또는 어드민에서 확인)
```

### Claude — 네이버 ↔ 어드민 연계 방식 + 테스트 계획 (2026-03-09)

**질문 1: 네이버 상품 정보를 자동 파싱해서 어드민 이벤트로 만들 수 있나?**

결론: **어드민에 따로 등록하고 연계하는 게 맞다.**

이유:
- 네이버 스마트스토어 Commerce API는 사업자 파트너 승인이 필요하고, 호출 제한도 있어서 실시간 연동이 무겁다
- 네이버 상품명은 `[광주] MelON(멜론) 지브리 & 뮤지컬 오케스트라 콘서트 비지정석 광주, VIP석` 같은 판매용 제목이고, 어드민 이벤트는 `공연명 + 공연장 + 시작시간 + 좌석배치도 + 포스터` 같은 운영 정보가 필요함
- 두 시스템이 담당하는 정보의 성격이 다름

**대신 연계를 쉽게 만드는 방법:**

어드민에서 이벤트를 등록할 때 **네이버 상품 키워드**를 같이 저장한다.
예: `naverKeywords: ["광주", "지브리", "뮤지컬"]`

그러면 텔레그램 주문 알림이 왔을 때:
```
공연명: "[광주] MelON(멜론) 지브리 & 뮤지컬 오케스트라 콘서트..."
  → naverKeywords에 "광주" + "지브리" 포함된 이벤트 검색
  → 자동 매칭!
```

이렇게 하면:
- 네이버에서 상품 등록은 네이버에서 하고
- 어드민에서 이벤트 등록은 어드민에서 하되
- **키워드 매칭으로 자동 연결**됨
- 수동으로 "이 네이버 상품 = 이 어드민 이벤트" 매핑할 필요 없음

**질문 2: 테스트는 어떻게?**

실제 주문 없이 테스트하는 방법:

1. **텔레그램 봇에 `/testorder` 명령어 추가**
   - 예시 주문 메시지를 봇이 직접 생성해서 파싱 → API 호출
   - `dryRun: true`로 실제 좌석 예약 없이 흐름만 검증

2. **텔레그램에서 직접 테스트 메시지 전송**
   - 예시 알림 메시지를 봇 채팅방에 직접 붙여넣기
   - 봇이 파싱 → `createNaverOrderHttp` (dryRun) → 결과 회신

3. **실제 발권 테스트**
   - 테스트용 이벤트를 어드민에 하나 만들고
   - 텔레그램에서 가짜 주문 메시지를 보내서
   - 실제로 모바일 티켓 링크가 생성되는지 확인
   - 링크 열어서 티켓 화면까지 확인

**실행 순서 제안:**

| 순서 | 작업 | 설명 |
|------|------|------|
| 1 | 메시지 파서 구현 | `parseNaverOrderMessage()` 함수 |
| 2 | 어드민 이벤트에 `naverKeywords` 필드 추가 | 이벤트 매칭용 |
| 3 | 봇에 주문 감지 + API 호출 로직 추가 | 📦 메시지 → 파싱 → 발권 |
| 4 | `/testorder` 명령어 추가 | dryRun 테스트용 |
| 5 | 테스트 이벤트로 실제 발권 테스트 | 링크 열어서 확인까지 |

> 승인하면 이 순서대로 실행할게!

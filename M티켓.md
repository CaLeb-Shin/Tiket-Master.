## 2. 게이트 운영 구성

```
[ 일반 입장 게이트 A ]  ← 종이 티켓 / 기존 방식
[ 일반 입장 게이트 B ]  ← 종이 티켓 / 기존 방식
[ M 티켓 전용 게이트 ]  ← QR 스캔 전용, 스태프 1~2명
```

### M 티켓 게이트에 필요한 것
- **스캐너 기기 1~2대** (스마트폰/태블릿, 사전 승인 완료)
- **와이파이 또는 LTE** 안정적 연결 (QR 검증은 서버 호출)
- **안내 표지판**: "M 티켓 입장 전용" (M 티켓 소지자만 이 줄)
- **비상용 노트북 1대**: 어드민 페이지 접속용 (수동 확인 백업)

## 3. 입장 흐름

```
관객 도착
  │
  ├─ 종이 티켓 소지자 → 일반 게이트 → 수동 확인 입장
  │
  └─ M 티켓 소지자 → M 티켓 게이트
       │
       ├─ 1. 스마트폰에서 M 티켓 URL 열기 (카톡/문자 링크)
       │
       ├─ 2. QR 코드 표시 (자동 2분마다 갱신)
       │
       ├─ 3. 스태프가 스캐너 앱으로 QR 스캔
       │     └─ 성공: 초록색 "입장 확인" + 좌석 정보 표시
       │     └─ 실패: 빨간색 에러 (만료/중복/취소 등)
       │
       ├─ 4. 성공 시 입장 허용
       │
       └─ 5. 실패 시 → 스태프가 수동 확인 (어드민 페이지)
```

### Claude — 추가 요구사항 구현 계획 (2026.03.24)

#### A. 세 가지 입장 방식 통합

```
공연장 입장
  │
  ├─ 1) 놀티켓 발권  ← 종이 티켓, 기존 수동 확인
  ├─ 2) 네이버 티켓 발권  ← 네이버 자체 QR/바코드
  └─ 3) M 티켓 (모바일)  ← 우리 시스템 QR 스캔
```

현재 M 티켓 파이프라인: 네이버 주문 → 어드민 입력 → M 티켓 자동 생성 → URL 발송 → QR 입장

**필요 기능:** 어드민에서 네이버 주문 목록 중 M 티켓 대상만 **선별/분리**하는 UI
- 주문 목록에 "입장 방식" 컬럼 추가 (놀티켓 / 네이버티켓 / M티켓)
- 입장 방식별 필터링
- M 티켓으로 전환 버튼 (선택한 주문 → M 티켓 발급)

#### B. 전체 플로우 테스트 환경

예매부터 공연 종료까지 시뮬레이션 가능한 **테스트 모드**:

| 단계 | 기능 | 구현 |
|------|------|------|
| 1. 예매 | 테스트 주문 생성 | 어드민 → 테스트 주문 입력 (이미 가능) |
| 2. 발권 | M 티켓 생성 + URL 발급 | 이미 구현됨 |
| 3. 좌석 공개 | revealAt 즉시 트리거 | 어드민에서 **"지금 공개"** 버튼 추가 필요 |
| 4. 입장 | QR 스캔 체크인 | 이미 구현됨 |
| 5. 인터미션 | 설문지 표시 | **신규 구현 필요** |
| 6. 공연 종료 | 네이버 리뷰 유도 | **신규 구현 필요** |

#### C. 인터미션 설문지 + 공연 후 네이버 리뷰

**M 티켓 화면 상태 전환 (어드민 제어):**

```
[공연 전]          → 좌석 + QR 표시
  ↓ (어드민: "입장 시작" 클릭)
[입장 중]          → QR 활성화
  ↓ (어드민: "인터미션 시작" 클릭)
[인터미션]         → 간단 설문지 표시 (별점 + 한줄평)
  ↓ (어드민: "2부 시작" 클릭)
[2부 진행 중]      → 좌석 정보 표시
  ↓ (어드민: "공연 종료" 클릭)
[공연 종료]        → 네이버 리뷰 작성 유도 카드 표시
```

### Claude — D-1, D-2 구현 완료 (2026.03.24)

**D-1 어드민 공연 진행 컨트롤 패널 ✅**
- `LivePhase` enum: pre → entry → intermission → part2 → ended
- `EventPhaseControlScreen`: 타임라인 형태 단계 전환 UI
- 어드민 대시보드 이벤트 메뉴에서 "공연 진행 제어" 진입
- Firestore 즉시 반영 + 확인 다이얼로그

**D-2 M 티켓 화면 상태 연동 ✅**
- `getEventLivePhase` CF 추가 (가벼운 폴링용)
- 모바일 티켓 페이지: 15초마다 livePhase 폴링
- `livePhase == 'intermission'` → 설문지 표시
- `livePhase == 'ended'` → 리뷰 유도 카드 표시

**D-3 인터미션 설문지 ✅** — 이미 구현되어 있음 (submitIntermissionSurvey CF + UI)
- D-2에서 livePhase == 'intermission'일 때 자동 표시 연동 완료

**D-4 공연 후 네이버 리뷰 유도 ✅** — 이미 구현되어 있음 (리뷰 카드 + confirmNaverReview CF)
- D-2에서 livePhase == 'ended'일 때 자동 표시 연동 완료

**D-5 주문별 입장 방식 선별 ✅**
- `EntryMethod` enum: paper(놀티켓) / naver(네이버) / mTicket(M 티켓)
- NaverOrder 모델에 entryMethod 필드 추가
- 어드민 주문 목록에 색상 배지 표시 + 탭으로 순환 전환

**D-6 revealAt 즉시 트리거 ✅**
- 공연 진행 제어 화면에 "좌석 + QR 지금 공개" 버튼
- 테스트 시 즉시 좌석/QR 공개 가능

**사실 현재 시스템이 이미 이 구조:**
- `totalEarned`: 누적 (감소 안 함) → 티어 판정용
- `balance`: 사용 가능분 → 업그레이드 시 차감
